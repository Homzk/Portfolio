# Contract — Internacionalización (ES/EN)

Capa propia sin librería externa (research D4). Un único estado global controla todo el
sitio (Constitución III; FR-007..FR-010a).

## API de contexto

`src/i18n/LangContext.jsx` expone:

```text
LangProvider            // envuelve la app por encima del RouterProvider
useLang() -> {
  lang: 'es' | 'en',    // idioma actual
  setLang(next),        // cambia y persiste en localStorage
  toggle(),             // alterna es<->en
  t(dict)               // helper: devuelve dict[lang]
}
```

## Reglas

- **R-I1 (default)**: primera visita sin valor en `localStorage` → `lang = 'es'` (FR-008).
- **R-I2 (persistencia)**: cada `setLang`/`toggle` escribe la clave única en `localStorage`;
  al cargar se lee de ahí (FR-010a).
- **R-I3 (degradación)**: si `localStorage` no está disponible, usar estado en memoria con
  default `es` sin romper.
- **R-I4 (un solo toggle)**: el control de idioma se renderiza únicamente en el `Nav` del
  Hero; ninguna otra parte muestra otro toggle (FR-007).
- **R-I5 (paridad)**: toda clave de cadena tiene `es` y `en`; un test de paridad falla si
  alguna falta (FR-009). El copy canónico proviene de `references/content.md`.

## Diccionarios

`src/i18n/strings.js` agrupa por dominio espejo de la referencia:
`HERO`, `PROJ` (por proyecto), `SEC` (About/Stack/Experience/Contact), `CASE` (plantilla
de caso de estudio). Cada hoja es `{ es, en }`.

## Criterios de aceptación

- AC-I1: Con `localStorage` vacío, el sitio carga en español.
- AC-I2: Tras togglear a EN y recargar, el sitio vuelve en inglés.
- AC-I3: Cambiar idioma actualiza Hero, todas las secciones y cualquier caso de estudio
  abierto, sin fragmentos en el idioma anterior.
- AC-I4: El test de paridad de claves pasa (0 cadenas huérfanas en ES o EN).
